🎯 Introducción

El análisis semántico es una fase fundamental del proceso de compilación. Después del análisis léxico y sintáctico, el compilador verifica que el programa tenga coherencia lógica y compatibilidad de tipos antes de generar código.

📌 1.1 Árboles de Expresiones

Representan la estructura jerárquica de una expresión matemática o lógica, respetando la precedencia de operadores.

a + b * c
  • Representan precedencia de operadores
  • Facilitan evaluación semántica
  • Base para generación de código intermedio
📌 1.2 Acciones Semánticas

Instrucciones que se ejecutan cuando el analizador sintáctico reconoce una producción gramatical.

  • Verificación de tipos
  • Construcción de tabla de símbolos
  • Generación de código intermedio

📌 1.3 Comprobación de Tipos
int a = 5 + 3; ✔ Correcto
int a = "hola" + 5; ✖ Error
  • Compatibilidad de tipos
  • Conversión implícita
  • Promoción de tipos
📌 1.4 Pila Semántica

Almacena identificadores, tipos y valores temporales durante el análisis LR. Permite mantener coherencia entre símbolos y tipos.

📌 1.5 Esquema de Traducción
E → E1 + T { E.valor = E1.valor + T.valor }

Combina reglas gramaticales con acciones semánticas para dar significado a cada producción.

📌 1.6 Tabla de Símbolos
x | int | global | 1000

Almacena información de identificadores, tipo, ámbito y dirección en memoria.

📌 1.7 Manejo de Errores
y = 10; // Error si no fue declarada
  • Variable no declarada
  • Tipos incompatibles
  • Asignaciones inválidas
🧠 Conclusión

El análisis semántico valida la coherencia lógica del programa antes de su ejecución. Gracias a estructuras como árboles de expresión, tablas de símbolos y pilas semánticas, el compilador puede detectar errores y preparar el código para su traducción a lenguaje máquina.
